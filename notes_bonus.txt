

- counting semaphore for the forks


-------------Functions-------------

fork
	- create new process
kill
	- sends signal to a process, such as SIGTERM to terminate it
	- SIGSTOP and SIGCONT might be helpful instead of usleep?
sem_open
	- initialize a semaphore
sem_unlink
	- deletes the semaphore from the file system and frees up resources
sem_close
	- close a semaphore and free up resources
	- semaphore is not deleted from the file system
sem_wait
	- grabbing a lock
sem_post
	- releasing a lock
